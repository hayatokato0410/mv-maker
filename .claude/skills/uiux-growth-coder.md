# uiux-growth-coder（Claude Code Skill）

このスキルは、UI/UXの実装タスクを「設計→実装→検証」の順で確実に前に進めるための実務手順です。
UI/UX Pro Max のワークフロー思想（デザインシステム生成→推奨→コード化→チェック）をベースに、UX心理学の原則と、Monetization awareness のパターンを**実装レベル**に落とし込みます。

---

## 使いどころ（自動起動の目安）
次の要求が来たらこのスキルを使う：

- 画面を作る（LP / ダッシュボード / 設定 / オンボーディング / 課金）
- UXを改善する（離脱、認知負荷、理解度、操作ミス、視線誘導）
- 収益化の認知・導線を改善する（Pro表示、機能ロック、アップグレード誘導）
- A/Bテストを想定してUIを分岐できる形で実装したい

---

## 実行フロー（6ステップ）
> 迷ったらこの順で進める。最後に「納品前チェック」で潰す。

### 1) 目的と制約を確定（最短で）
- 目的：KPI（例：CVR、無料→有料転換、アクティベーション、継続）
- 対象ユーザー：主要セグメント（初心者/熟練、無料/有料）
- 画面スコープ：どのページ/フローを触るか
- 技術スタック：HTML+Tailwind / React(Next.js, shadcn) / Vue / Svelte / SwiftUI / Compose / RN / Flutter
- 計測：イベント名、プロパティ、成功指標、ガードレール（解約/不満/CS）

### 2) デザインシステム（局所でもOK）を作る
- まず「最小のデザインシステム」を文章化：色、タイポ、余白、角丸、影、コンポーネントの型
- 既存プロダクトがあれば、現状に寄せる（勝手に刷新しない）
- 参考：UI/UX Pro Max の「Design System Generated」思想

※ 詳細手順は [UI_UX_PRO_MAX_WORKFLOW.md](UI_UX_PRO_MAX_WORKFLOW.md)

### 3) UX心理学の原則で「設計判断」を補強
- "きれい"だけでなく、意思決定と行動を前に進める
- 使う原則は「目的に効くものだけ」を選ぶ（過剰適用しない）

※ 原則一覧と適用パターンは [UX_PSYCHOLOGY.md](UX_PSYCHOLOGY.md)

### 4) Monetization awareness（課金認知）をUIに入れる
- 無料ユーザーが「どこが有料か」を迷わない状態を作る
- 代表パターン：Proバッジ／機能ロック（コンテンツロック）

※ 実装ガイドは [MONETIZATION_AWARENESS.md](MONETIZATION_AWARENESS.md)

### 5) コード実装（A/Bテストしやすい構造で）
- UIはコンポーネント化（ProBadge, LockedFeatureCard, UpgradeCTA, PaywallEntry など）
- 変更多発ポイントはフラグ化（Feature Flag / Remote Config）
- 計測を同時に入れる（表示、クリック、到達、成功、キャンセル）
- アクセシビリティとレスポンシブは初手で担保（後回しにしない）

### 6) 納品前チェック（落とし穴を潰す）
- 視覚的階層：主CTAが最短で目に入るか
- 認知負荷：文言/要素が多すぎないか
- バナー・ブラインドネス：重要情報が広告っぽく見えて無視されないか
- "課金サプライズ"回避：無料でできると思わせて突然の壁になってないか（Proバッジ等で予告）
- 例外状態：権限なし/未ログイン/ネット不良でも壊れないか
- トラッキング：イベント名・プロパティが一貫しているか

## 出力フォーマット（このスキルの成果物）
要求に応じて、次をセットで出す：

1. **実装プラン**（目的/KPI・対象画面・心理原則・monetizationパターン・計測）
2. **UI構造**（コンポーネント一覧、状態、Props、バリアント、フラグ）
3. **コード**（必要ファイルと差分、スタイル方針）
4. **テスト観点**（UI/UX、a11y、E2E、計測）
5. **A/Bテスト案**（仮説、一次指標、ガードレール、サンプルイベント）
